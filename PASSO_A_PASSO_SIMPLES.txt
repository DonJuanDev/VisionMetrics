┌─────────────────────────────────────────────────────────────┐
│ ERRO: "Database connection error"                          │
└─────────────────────────────────────────────────────────────┘

Causa: Banco não foi importado

Solução:
1. Vá no phpMyAdmin
2. Clique no banco "visionmetrics"
3. Verifique se existem tabelas criadas
4. Se não, importe sql/schema.sql novamente


┌─────────────────────────────────────────────────────────────┐
│ ERRO: "500 Internal Server Error"                          │
└─────────────────────────────────────────────────────────────┘

Causa: Problema no .htaccess

Solução:
1. Via FTP, renomeie .htaccess para .htaccess_backup
2. Teste o site
3. Se funcionar, o problema estava no .htaccess


┌─────────────────────────────────────────────────────────────┐
│ ERRO: "Class not found"                                    │
└─────────────────────────────────────────────────────────────┘

Causa: Pasta vendor/ não foi enviada

Solução:
1. Execute: composer install --no-dev
2. Faça upload da pasta vendor/ completa


┌─────────────────────────────────────────────────────────────┐
│ ERRO: Página em branco                                     │
└─────────────────────────────────────────────────────────────┘

Causa: Erro de PHP oculto

Solução:
1. Edite o arquivo .env via FTP
2. Mude: APP_DEBUG=false para APP_DEBUG=true
3. Recarregue a página para ver o erro
4. Volte para false depois!
